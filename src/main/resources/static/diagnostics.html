Common Issues: 1. Connection Refused - Ensure Proxmox/Guacamole servers are running - Check firewall rules allow connections - Verify URLs in application-local.yml 2. 401 Unauthorized (Proxmox) - Verify token-id format: username@realm!token-name - Example: root@pam!cloud-desktop-token - Check token secret is correct - Ensure API token has proper permissions 3. 401 Unauthorized (Guacamole) - Verify username and password - Default is guacadmin/guacadmin - Check Guacamole is properly configured 4. 404 Not Found - Verify the service URL is correct - Check the service is accessible at that URL Configuration File: Edit: src/main/resources/application-local.yml Run with: ./mvnw spring-boot:run -Dspring-boot.run.profiles=local
